# Firebase 유저 데이터 관리 시스템 - 구현 설명서

## 1. 구현한 기능 목록

### 기본 기능
- 회원가입 (`UserRegister.cs`)
  - 닉네임 빈 값 검사
  - 닉네임 중복 검사 (`OrderByChild("NickName").EqualTo(...)`)
  - 회원가입 성공 시 Firebase `UserInfo`에 신규 유저 데이터 저장
  - 회원가입 성공 시 `UserKey`, `UserNickName`을 `PlayerPrefs`에 저장
- 로그인 (`UserLogin.cs`)
  - 닉네임 빈 값 검사
  - 존재하지 않는 닉네임이면 로그인 실패 처리
  - 로그인 성공 시 `UserKey`, `UserNickName`을 `PlayerPrefs`에 저장
- 코인 / 인벤토리 불러오기 (`ShopManager.cs`, `InventoryManager.cs`)
  - `PlayerPrefs`의 `UserKey`로 본인 데이터만 조회
  - `Coin`, `Inventory`, `UnitList`를 Firebase에서 로드
- 아이템 구매 (`ShopManager.cs`)
  - 코인 부족 시 구매 차단
  - 구매 성공 시 코인 차감 + 인벤토리 개수 증가 후 Firebase에 저장
- 인벤토리 표시 (`InventoryManager.cs`)
  - `Inventory` JSON을 `Dictionary<string,int>`로 변환하여 아이템별 개수 표시
- 아이템 사용 (`InventoryManager.cs`)
  - 개수가 0이면 사용 불가
  - 사용 시 개수 1 감소 + Firebase 저장 + 아이템별 사용 메시지 출력

### 심화 과제 (모두 구현)
- 심화 1. 유닛 구매 기능 (`ShopManager.cs`)
- 심화 2. 게임 결과 저장 (`GameResultManager.cs`)
- 심화 3. 거래소 / 경매장 기능 (`MarketManager.cs`, `MarketItem.cs`)

## 4. Firebase 데이터 구조 설명

```
UserInfo
 └── 유저고유키 (Push Key)
      ├── NickName : string
      ├── Coin : int
      ├── Score : int
      ├── UnitList : string (JSON, Dictionary<string,bool>)
      └── Inventory : string (JSON, Dictionary<string,int>)

Market   (심화 3. 거래소)
 └── 등록키 (Push Key)
      ├── SellerKey : string   (판매자의 UserInfo 키)
      ├── ItemName  : string
      ├── Price     : int
      └── Status    : string  ("OnSale" | "Sold")
```

회원가입 시 기본 데이터 예시:

```json
{
  "NickName": "Player01",
  "Coin": 500,
  "Score": 0,
  "UnitList": "{\"Unit1\":true,\"Unit2\":false,\"Unit3\":false,\"Unit4\":false}",
  "Inventory": "{\"Pistol\":0,\"Shotgun\":0,\"SniperRifle\":0,\"RocketLauncher\":0}"
}
```

## 5. PlayerPrefs에 저장한 값

| 키 | 값 | 저장 시점 |
|---|---|---|
| `UserKey` | Firebase `UserInfo` 하위 유저 고유 Push Key | 회원가입 성공 / 로그인 성공 시 |
| `UserNickName` | 입력한 닉네임 | 회원가입 성공 / 로그인 성공 시 |

이후 모든 화면(`ShopManager`, `InventoryManager`, `GameResultManager`, `MarketManager`)은 `PlayerPrefs.GetString("UserKey")`로 본인 데이터 경로(`UserInfo/{UserKey}`)만 읽고 씁니다. 이를 통해 "로그인 후 다른 유저 데이터가 불러와지는" 문제를 방지합니다.

## 6. Inventory JSON 처리 방식

- `Inventory`, `UnitList`는 Firebase에는 **JSON 문자열**로 저장됩니다.
- 읽을 때: `JsonConvert.DeserializeObject<Dictionary<string,int>>(json)` / `Dictionary<string,bool>`로 변환하여 메모리에서 사용
- 수정할 때: 메모리의 `Dictionary`를 직접 증감(`inventory[item]++` 등) 처리
- 저장할 때: `JsonConvert.SerializeObject(inventory)`로 다시 JSON 문자열로 변환하여 `SetValueAsync` 또는 `UpdateChildrenAsync`로 Firebase에 반영
- 거래소(`MarketManager`)의 `Market/{key}` 항목은 `MarketItem` 클래스를 통째로 `SetRawJsonValueAsync` / `GetRawJsonValue()`로 직렬화·역직렬화합니다.

## 7. 심화 과제 설명

### 심화 1. 유닛 구매 기능
- `ShopManager.cs`에서 `UnitList`를 함께 로드
- `OnClickBuyUnit2/3/4()` → `BuyUnit(unitName)`
  - 이미 보유(`unitList[unitName] == true`)면 구매 차단
  - 코인 부족 시 구매 차단
  - 통과 시 코인 차감 + `unitList[unitName] = true` 후 `Coin`, `UnitList` 저장

### 심화 2. 게임 결과 저장
- `GameResultManager.cs`의 `EndGame(score, rewardCoin)`
  - Firebase에서 현재 `Coin`, `Score`를 로드
  - `Coin += rewardCoin`
  - `Score = Max(현재 Score, 이번 score)` → 기존 점수보다 높을 때만 갱신
  - 결과를 `UpdateChildrenAsync`로 저장

### 심화 3. 거래소 / 경매장 기능
- `Market` 노드에 판매 등록 항목을 `Push()`로 추가 (`MarketManager.ListItem`)
  - 판매 등록 시 본인 인벤토리에서 해당 아이템 1개를 즉시 차감 (에스크로 방식 → 중복 판매 방지)
- `LoadListings()`로 `Status == "OnSale"`인 항목만 목록에 표시
- `BuyListing(marketKey)` (`MarketManager.BuyListingFromBuyer` → `PaySeller` → `CompletePurchase`)
  - 구매자: 코인 확인 → 코인 차감 + 인벤토리 증가
  - 판매자: 코인 증가
  - 거래 완료 후 `Market/{key}/Status`를 `"Sold"`로 변경하여 재구매 방지

## 8. 실행 중 발생한 문제와 해결 방법

### 문제 1. 인벤토리/거래소 씬에서 버튼 클릭이 동작하지 않음

- **증상**: OnClick 이벤트가 연결되어 있고 MessageText도 정상 연결되어 있음에도 버튼 클릭 시 아무 반응 없음
- **원인**: 해당 씬에 `EventSystem` 오브젝트가 없었음. Canvas를 수동으로 추가할 때 EventSystem이 자동 생성되지 않아 버튼 입력이 전달되지 않음
- **해결**: Hierarchy 우클릭 → `UI` → `Event System` 추가 후 정상 동작 확인

---

### 문제 2. Firebase SDK DllNotFoundException 오류

- **증상**: Play 모드 실행 시 `DllNotFoundException: FirebaseCppApp-13_11_0` 오류 발생, Firebase 기능 전체 중단
- **원인**: git 명령 실행 중 `Assets/Firebase/Plugins/x86_64/` 폴더의 네이티브 플러그인 바이너리 파일이 로컬에서 삭제됨
- **해결**: `Assets` → `Import Package` → `Custom Package`에서 `FirebaseDatabase.unitypackage` 재임포트하여 복구

---

### 문제 3. GitHub 대용량 파일 push 거부 오류

- **증상**: `git push` 시 `GH001: Large files detected` 오류 발생. Firebase SDK 네이티브 파일(최대 107MB)이 GitHub 제한(100MB) 초과
- **원인**: Firebase SDK의 x86_64 플러그인 폴더가 git에 추적되어 커밋에 포함됨
- **해결**: `.gitignore`에 `Assets/Firebase/Plugins/x86_64/` 추가, `git rm -r --cached`로 추적 해제 후 push
